  Model Checking vs Theorem Proving — The Answer

  Use both, but for different things:

  Model checking (herd7/TLA+) is better for:
  - Exploring litmus tests exhaustively (the litmus_tests.hip file)
  - Finding bugs in specific fence sequences you haven't thought of
  - Handling the interleaving explosion (N threads × M instructions × K addresses)
  - Validating against real hardware behavior

  Theorem proving (what we just did in Lean) is better for:
  - Proving optimizations are sound for all states (universal quantification)
  - The structural insight: release and acquire touch disjoint state (fence_commute_final_state) — this is WHY coalescing works, not just that it works for specific test cases
  - Scope narrowing — this is fundamentally a universal proof ("for ALL stores and ALL loads, chiplet fence suffices within same XCD"). A model checker can only verify this for bounded
  instances.
  - Compositionality — hierarchical_correctness says the two-level approach works for ANY topology. This composes the chiplet and device proofs.

  The practical workflow:
  1. Write litmus tests, run on hardware (litmus_tests.hip)
  2. Model check with herd7/TLA+ to explore interleavings
  3. Once confident in the model, prove structural properties in Lean (optimizations, scope narrowing, composition)
  4. The Lean proofs give you confidence that the optimizations are correct by construction, not just "we didn't find a bug yet"
